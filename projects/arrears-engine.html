Tenant Arrears Automation Engine

A property operations automation system designed to convert spreadsheet-based tenant ledgers into reliable arrears intelligence, reminder outputs, and operational dashboard visibility.

Business Problem

Manual arrears tracking in property portfolios often breaks down when partial payments, mixed charge histories, and inconsistent follow-up processes are involved.

Teams end up with unreliable aging buckets, poor visibility into outstanding balances, and reminder workflows based on distorted information.

System Built

Lionyx-E designed a structured arrears engine that accepts Excel-based tenant and ledger inputs, applies FIFO allocation logic, calculates aging positions, and produces operational outputs for follow-up.

Tenant & Ledger Inputs (Excel) ↓ Validation Layer ↓ Python Arrears Engine ↓ FIFO Allocation Logic ↓ Arrears Snapshot Output ↓ Reminder Output + Dashboard

Core Logic

The engine uses FIFO allocation to apply payments to the oldest outstanding charges first, which produces more reliable arrears aging than loose spreadsheet methods.

Operational Outputs

The system generates snapshot-style outputs that can feed dashboards, follow-up workflows, and management reporting with less manual reconciliation.

Business Outcome

The result is cleaner arrears visibility, faster follow-up, and a more reliable operational picture of portfolio health.

Technology Structure

This system was designed as an Excel + Python operational architecture:

  • Excel for familiar operational input
  • Python for processing logic and allocation rules
  • CSV / report outputs for downstream dashboard use
  • A structure suitable for recurring operational workflows
